    Info

    Documentation de l'API de news.capstan.fr.

    Code HTTP

    Les codes HTTP utilisés sont les plus communs.

    Code Signification
    200 OK
    201 Élément créé
    401 L'utilisateur n'est pas connecté
    404 Page introuvable
    422 Erreur de validation (voir messages)
    500 Erreur serveur
    503 Serveur en maintenance

    Messages d'erreurs

    {
    "messages": [
        "Vous devez être connecté"
    ]
}

    Les messages d'erreurs peuvent être récupérés via l'attribut messages qui est un tableau. Les messages d'erreurs sont en français et peuvent être directement affichés à l'utilisateur dans le cadre de messages de validation (code 422) par exemple.

    Articles

    Lister les articles

    Cette URL permet de lister les articles paginés, trier par date de publication.

    Concernant le paramètre favorites, il n'est pas nécessaire de lui donner une valeur. Seule sa présence ou non est prise en compte. Si aucun utilisateur n'est connecté avec ce paramètre, l'API répondra avec un message d'erreur.

    L'attribut content contient le contenu HTML de l'article. L'HTML peut contenir des balises p, h2, a, strong, em, s, u, ol, ul, li. Il peut également contenir des div.style-quote qui représentent un bloc de citation ainsi que des div.style-framed qui représente un bloc gris de mise en valeur.

    Le format d'un article est sa mise en valeur, voir tableau de correspondance suivant :

    Clé Nom
    default Classique
    featured En vedette
    framed Encadré
    video Vidéo

    Pour le paramètre author_capstan_id, il fait référence à l'identifiant unique d'un auteur sur le site capstan.fr que les administrateurs devront renseigner manuellement dans l'administration du site Capstan News.

    Example request:

    curl -X GET \
    -G "https://news.capstan.fr/api-capstan/articles" \
    -H "Content-Type: application/json" \
    -H "Accept: application/json" \
    -H "Capstan-Authorization: 123456"
    
$client = new \GuzzleHttp\Client();
$response = $client->get(
    'https://news.capstan.fr/api-capstan/articles',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
            'Capstan-Authorization' => '123456',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));

    Authentification

    Clé d'API utilisateur

    Cette URL permet d'afficher la clé d'authentification unique de l'utilisateur grâce à ses informations de connexion

    Cette clé d'API est unique pour chaque utilisateur et permet de l'authentifier pour les requêtes où c'est nécessaire en rajoutant le header Authorization: Bearer *clé d'API*

    La clé d'API doit rester strictement confidentielle et ne doit pas être dévoilée à l'utilisateur pour des raisons de sécurité

    Example request:

    curl -X POST \
    "https://news.capstan.fr/api-capstan/auth/api-token" \
    -H "Content-Type: application/json" \
    -H "Accept: application/json" \
    -H "Capstan-Authorization: 123456" \
    -d '{"email":"webmaster@weblogin.fr","password":"123456"}'

    
$client = new \GuzzleHttp\Client();
$response = $client->post(
